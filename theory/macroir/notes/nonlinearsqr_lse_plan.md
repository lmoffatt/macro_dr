## 1. Math
- Mean (deterministic, non-recursive): μ_i = N·(P_mean_i · gmean_i) + baseline. Single amplitude I_max = N·g (N and g unidentifiable from the mean → fix one).
- SSE = Σ_{finite i} (y_i − μ_i)². n = count of finite y_i.
- logL = lgamma(n/2) − (n/2)·log(π) − (n/2)·log(SSE)   [Jeffreys marginal; IMPROPER prior → defined up to a constant c (§8), so NOT cross-family comparable; θ-inference is c-invariant → exact for the eLife deliverable. Constant convention c=1. Derivation checked: the (2π)^(−n/2) and the Γ-integral 2^(n/2) cancel to π^(−n/2), NOT 2π.]
- score = ∂logL/∂θ = −(n/(2·SSE))·dSSE,  dSSE = −2·Σ (y_i−μ_i)·dμ_i  → score = (n/SSE)·Σ(y_i−μ_i)dμ_i (signs verified)
- Fisher: the code reports the classical NLS plug-in `(n/SSE)·Σ dμ_i dμ_iᵀ = (1/σ̂²)·JᵀJ`, σ̂²=SSE/n (the GN-Hessian at the optimum; standard NLS CI). METHODS wording ([4] — do NOT label the (n/SSE) form "the exact expected information"): the POPULATION expected info at the true θ IS exactly (1/σ²)·JᵀJ (profile-Hessian corrections n/(n−2) and −2/(n−2) cancel to 1 — verified, comms-log; so the review-thread "carries (n−2)/n" is about the PLUG-IN, not the population quantity). (n/SSE) is the ML plug-in of it (biased up by n/(n−2); unbiased plug-in = (n−2)/SSE; gap O(1/n), negligible for the large per-recording n). score+Fisher are a consistent Newton pair; if logL is −(n/2)·log(·) on a `Derivative<SSE>` scalar the AD chain yields the score for free. CAVEAT: this is the naive model-based (homoscedastic) info; the exact simulator's residual variance is heteroscedastic (gating N·gSg, §7) so a sandwich CI differs — fine for the "LSE validity vs exact sim" frame, state it.
- Parallel to macroir: macroir marginalizes the within-interval state trajectory; LSE marginalizes the noise scale.

## 2. Architecture (Option 2)
Dedicated LSE driver `nonlinearsqr_logLikelihood<...>` (value + derivative overloads), structured like `log_Likelihood` (qmodel.h:6693) but with a **lean per-interval body**:
- Reuse the shared `calc_Qdt` (kinetic heavy-lifting, algorithm-agnostic) exactly as the macro driver does.
- Per interval: propagate P_mean only (`r_P_mean = to_Probability(p_P_mean·t_P)`, qmodel.h:4408 pattern — P_Cov-independent, verified), compute μ_i and, in the derivative path, dμ_i (from `Derivative<P_mean>`). NO P_Cov, NO y_var, NO gSg, NO Kalman down-date.
- Accumulate additively: SSE, dSSE, Σdμ_i dμ_iᵀ, n. (SSE/dSSE ride inside one `Derivative<>` scalar; Σdμdμᵀ is the GN block.)
- One post-fold **finalize** (§5) builds the reduced state the optimizer + writer consume.

Duplication is small: the fold boilerplate + the lean step. The expensive `calc_Qdt` is reused, not reimplemented. Prefer a dedicated lean driver over parameterizing `log_Likelihood`'s step (its body carries test-derivative scaffolding not wanted here).

## 5. Finalize (pure fn of SSE, dSSE, Σdμdμᵀ, n)
- Rebuild `Derivative<logL>` with **value = logL (marginal, §1)** AND **derivative = score (§1)** together — the optimizer reads `get<Grad>` DERIVED from `Derivative<logL>` (no standalone Grad slot in `dMacro_State_Hessian_minimal`, verified). If only the value is rewritten, every Newton step is garbage (verify:completeness blocker).
- **AD caveat (Fisher is NOT the AD Hessian — user flag):** the score comes for free from FIRST-order AD if SSE rides the fold as a `Derivative<>` (logL=−(n/2)log(SSE_deriv) auto-carries dlogL=(n/SSE)Σr dμ). BUT the Fisher must NOT be `derivative(derivative(logL))` (second-order AD) — that is the TRUE Hessian −∂²ℓ_p = (n/SSE)JᵀJ − (n/SSE)Σr_i∂²μ_i − (2n/SSE²)(Jᵀr)(Jᵀr)ᵀ, whose last two terms vanish ONLY at a perfect fit / exactly at the optimum. Assemble the WANTED GN/expected block explicitly from the FIRST-order AD Jacobian rows (like the macro GFI `sqr_X<true>(derivative(get<y_mean>))` @6096, but WITHOUT the /y_var and WITHOUT the (dy_var)² term).
- Set `Gaussian_Fisher_Information` = (n/primitive(SSE))·Σ sqr_X(derivative(μ_i)). TWO differences from macro: (i) NO variance term (LSE has no y_var); (ii) ONE global σ̂²=SSE/n scales the whole JᵀJ post-fold (macro divides per-step by its own y_var — heteroscedastic). Use **primitive(SSE)** for the 1/σ̂² prefactor — evaluate it at the current θ, do NOT let AD differentiate the prefactor.
- **SSE→0 guard** (perfect fit / degenerate) mirroring existing isfinite/positive checks.
- Emit the reduced state the entrypoint expects: `Macro_State_reg`/logLs (value) and the Derivative-based `dMacro_State_Hessian_minimal` (MLE/derivative — the macrodr:: one @qmodel_types.h:1469 carrying Derivative<logL>, NOT the unrelated global one @distributions.h:1139 that lists an explicit Grad). ### Parked for the subsequent (program) paper: evidence & Bayes factors
Decided 2026-07-18: the LSE-vs-macro Bayes factor belongs to the subsequent paper, NOT eLife. Open discussion to close then. Points to carry over:
- Jeffreys prior on σ² is **improper** → marginal p(y|θ) = c·Γ(n/2)·π^(−n/2)·SSE^(−n/2) with arbitrary c → Z_LSE and BF Z_LSE/Z_macro defined only up to c → **ill-defined** without a proper σ² prior.

- Deeper: the BF really tests "is the residual scatter white noise or does it carry macro's gating signature" — meaningful but prior- and misspecification-sensitive; not a clean nested test.
- **θ-inference (point + Fisher/CI, the eLife deliverable) is invariant to c → unaffected.** Implement it now; decide the σ² prior later.
- Remedies to weigh then: proper IG(a,b) on σ²; fractional/intrinsic Bayes factor (data-split, removes c); or skip the BF and compare via exact-sim validity (the eLife method).
