# Scientific Software Project

## Identity

- Project id: `scientific-software`
- Program: `macroir`
- Type: theory / methodology / conceptual engineering
- Status: drafted
- Publication relevance: optional
- Primary output type: theory, conceptual structure, and reusable principles

## Purpose

Develop and curate the scientific-software theory that functions as part of the
conceptual foundation of the MacroIR / MacroDR program.

This includes work on ideas such as:

- proof-oriented design
- algebraic CLI and command-language structure
- provenance-aware design
- safety and contagion concepts
- broader scientific software design principles

## Scope

### In scope

- identify and curate the scientific-software theory documents already present
  in the repository
- distinguish baseline scientific-software theory from exploratory notes and
  archive
- provide a project home for this line of work
- support future promotion of relevant material into
  `theory/scientific-software/`
- support future paper or essay outputs derived from this line of work

### Out of scope

- reorganization of the whole repository
- MacroIR-specific mathematical derivations that belong under MacroIR theory
- implementation refactors that are not driven by this conceptual line of work

## Key problem

The repository contains an important body of scientific-software theory, but
that material is currently mixed with code-facing docs, theory drafts, and
paper-adjacent writing.

Without an explicit project home, it is harder to:

- track this work as an active line of development
- decide which documents are baseline-facing
- decide which documents belong to future papers or archives

## Deliverables

### D1. Project definition

A tracked project home for the scientific-software workstream.

### D2. Initial document classification

An initial classification of candidate scientific-software documents into:

- baseline-facing theory
- active notes
- archive
- paper-facing material

### D3. First promoted theory set

A first curated set of scientific-software documents promoted into
`theory/scientific-software/`.

## Success criteria

This project is successful if:

1. The scientific-software workstream has an explicit project identity.
2. Its documents are distinguishable from MacroIR-specific theory.
3. A first curated subset can be promoted into
   `theory/scientific-software/`.
4. The project can later develop internal subprojects or workstreams without
   losing coherence.
