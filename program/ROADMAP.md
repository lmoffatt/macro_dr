# MacroIR / MacroDR Roadmap

## Current roadmap scope

This roadmap is intentionally bounded.

Its current horizon is:

- bring MacroIR validation to the point of a validation paper
- clarify what is complete, what remains to be run, and what deliverables must
  be produced next

It does not try to define the full long-term future of the program.

## Program objective

The current primary objective of MacroIR is:

> validate a recursive macroscopic algorithm that can determine the evidence of
> alternative kinetic models from patch-clamp currents generated by many
> channels

More concretely, the current focus is the MacroIR family already developed in
the repository, especially the interval-time recursive formulation.

## Current source of truth

The current roadmap should be interpreted using the following source-of-truth
materials:

- active manuscript draft sources under
  `papers/macroir-elife-2025/docs/manuscript-drafts/`
- curated MacroIR theory documents under
  `theory/macroir/docs/Macro_IR/`
- validation-relevant Information Distortion Matrix documents under
  `theory/macroir/docs/Information_Distortion_Matrix/`

These sources take precedence over older informal notes when they disagree.

## Additional roadmap objective

Besides finishing the validation paper path, the roadmap should also document:

- the mapping between the current theory documents and the implemented
  algorithms
- which validation-relevant documents are canonical
- which validation-relevant documents are archival, superseded, or only
  historical support material

This is necessary because the algorithms already exist in implemented form, but
the corresponding theory and manuscript materials are distributed across
several document families.

## What is already complete

The current assessment is:

- the main MacroIR theory has been developed
- the code implementing that theory exists
- the validation theory needed to assess the code exists
- the validation-oriented code also exists

In other words, the present bottleneck is no longer foundational theory or the
first implementation. The bottleneck is empirical validation with concrete
examples and conversion of those results into a paper-quality output.

## Main remaining path

### Milestone 1. Define and finalize validation examples

Confirm the real validation examples that should be used to test MacroIR in
practice.

Outputs:

- agreed validation example set
- explicit run targets
- explicit success criteria for the examples

### Milestone 2. Run the validation examples

Execute the code on the selected examples.

Outputs:

- completed runs
- stored outputs
- reproducible run descriptions

### Milestone 3. Analyze the results

Turn raw runs into interpretable validation results.

Outputs:

- result summaries
- comparisons across examples or model alternatives
- diagnosis of where MacroIR works well and where it fails or remains uncertain

### Milestone 4. Produce figures for the validation paper

Convert analyzed outputs into paper-facing figures and tables.

Outputs:

- figure set
- captions or figure notes
- stable result selection for publication

### Milestone 5. Produce the MacroIR validation paper

Write the paper that validates MacroIR on the chosen examples.

Outputs:

- paper narrative
- figures integrated into the manuscript
- explicit statement of validation claims and limits

## Dependency structure

The current dependency chain is:

1. theory and implementation baseline
2. validation examples
3. executed runs
4. analyzed results
5. figures
6. validation paper

This means the immediate critical path is:

- choose or confirm the real examples
- run them
- analyze the outputs

The paper depends on these steps rather than the other way around.

## What is not on the current critical path

These areas remain important, but they are not the main bounded objective of
the current roadmap:

- future MacroIR extensions beyond the present validation target
- broader next-generation MacroDR redesign
- later papers beyond the validation paper
- new application areas that require further theory development

They should remain visible as future directions, but they should not displace
the current validation path.

## Future directions after the current roadmap

After the validation paper milestone, likely next directions include:

- new MacroIR algorithmic developments
- further characterization and benchmarking
- broader MacroDR refactor and infrastructure improvements
- additional applications and papers

Those should be treated as later roadmap phases or separate projects.

## Current practical priority

The current roadmap priority is:

1. confirm the validation examples
2. run them
3. analyze the results
4. generate figures
5. complete the validation paper

## Review rule

This roadmap should be revised whenever one of the following changes:

- the validation target changes
- the example set changes
- the current bottleneck moves from execution to theory or implementation
- the current paper scope changes
